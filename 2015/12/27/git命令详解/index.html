git命令详解

2015-12-27

作为一名大四在校生，做过仅有的一次团队开发中使用的是SVN,相对于分布式的版本控制系统Git还是有它的独特之处，初学git还是遇到了好多问题;
在windows上不用装虚拟机的情况下使用liux命令操作git给我带来一大挑战，也是一种极大的兴趣，对将来的工作肯定会有很大的帮助！

现在我所有个人项目都可以全都部署到github上了，并且相信现在大部分公司的项目迁移到git已是大势所趋！

切入正题：

Git是什么?

Git是一款免费、开源的分布式版本控制系统(CVS)，用于敏捷高效地处理任何或小或大的项目。

使用Git极大地提高了开发效率、扩大了开源项目的参与度、 增强了版本控制系统的安全性，

对于企业用户来说，使用Git早已是大势所趋！

Git有哪些优势？

  • 版本库本地化，支持离线提交，相对独立不影响协同开发。
  • Git对于每个工程只会产生一个.git目录，这个工程所有的版本控制信息都在这个目录中，不会像SVN那样在每个目录下都产生.svn目录。
  • 把内容按元数据方式存储，完整克隆版本库。 所有版本信息位于.git目录中，它是处于你的机器上的一个克隆版的版本库，它拥有中心版本库上所有的东西，例如标签、分支、版本记录等。
  • 支持快速切换分支方便合并，比较合并性能好。在同一目录下即可切换不同的分支，方便合并，且合并文件速度比SVN快。
  • 分布式版本库，无单点故障，内容完整性好。

常见Git命令使用

1、克隆到本地

  • git clone https://github.com/douhn/test.git

把远程库完整的拷贝一份到本地变成本地库，克隆到本地后所有的操作都针对本地库。当操作完成后推送到远程，即可把本地的更新同步到服务器

2、更新

  • git pull

在操作本地库前需要确保本地库的文件都是最新的。如果别人已经更新了远程库，而自己的本地库还是老代码，在提交时会出现错误。

2、查看状态

  • git status

可以列出当前目录所有还没有被git管理的文件和被git管理且被修改但还未提交(git commit)的文件查看本库下哪些文件没放入缓存区，哪些文件需要从暂存区删除

3、添加文件

  • git add

在git本地库文件夹新建个文件或者文件夹后并没有自动被git监管，需要把新添加的文件放入缓存区

4、提交到本地

  • git commit -m “提交备注”

提交的时候必须要输入提交描述。可以用git commit命令，会自动打开默认编辑器，在打开的文件下写入描述信息，保存。 也可以直接git commit -am “第一次提交”

在推送之前一定要再次执行 git pull,可能在你改动的这段时间里又有人推送过远程库。如果有人推送过，那么执行git pull的时候会自动把别人更新的代码合并到本地。此时需要再次执行git commit

5、删除文件

  • git rm file

在git本地库文件夹新建个文件或者文件夹后并没有自动被git监管，需要把新添加的文件放入缓存区

6、恢复删除的文件

  • git checkout commitid – file

7、修改文件

修改文件直接在文件修改后先git add . ， 再git commit即可。
也可直接git commit -am “message”

8、推送到远程

  • git push -u origin master

    把本地库推送到远程库

团队开发

1、分支操作

分支管理是git最重要特性，默认的分支是主干master

  • git branch: 查看当前所在的分支

  • git branch 分支名: 在本地库创建新的分支

  • git branch -a: 查看本地库和远程库的所有分支

  • git push -u origin 分支名: 把本地库创建的分支推送到远程库

  • git checkout 分支名: 切换到新的分支

2、代码合并

一般的开发在分支开发，当在分支开发完成要发布时需要把分支的代码合并到主干。
先切换到master分支， 然后执行git merge.

  • git merge 分支名: 把分支名里的内容合并到当前分支

3、Log查看

  • git log –oneline 紧凑查看

  • git log –graph 图形显示

  • git log erlang ^master 查看只在erlang分支里的修改

  • git log –grep 正则取一个log

  • git shortlog master 生成一个简报
